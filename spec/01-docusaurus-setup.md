## LanguageTabs Component

File: `website/src/components/LanguageTabs.tsx`

A reusable React component wrapping Docusaurus `<Tabs>`.

### Props

```typescript
interface LanguageTabsProps {
  children: React.ReactNode;  // Tab content blocks
}
```

### Behavior

- Renders a `<Tabs>` component with `groupId="language"` and `queryString="lang"`
- Tab labels: `Rust`, `TypeScript`, `Go`, `Python`, `PHP`
- Tab values: `rust`, `typescript`, `go`, `python`, `php`
- Selecting a tab persists across ALL `LanguageTabs` instances on the page (Docusaurus built-in behavior via `groupId`)
- Default tab: `rust`

### Usage in MDX

```mdx
import LanguageTabs from '@site/src/components/LanguageTabs';
import TabItem from '@theme/TabItem';

<LanguageTabs>
<TabItem value="rust">

\`\`\`rust
let node = CairnNode::builder().build().await?;
\`\`\`

</TabItem>
<TabItem value="typescript">

\`\`\`typescript
const node = await CairnNode.create();
\`\`\`

</TabItem>
<!-- ... other languages -->
</LanguageTabs>
```

### Implementation Note

Must work correctly with Docusaurus MDX v3 parser. The `groupId` synchronization is a built-in Docusaurus feature but requires correct `<TabItem>` nesting.

## Build and Deploy Commands

### Development

```bash
cd website && npm install && npm start
```

### Production Build

```bash
cd website && npm ci && npm run build
# Output: website/build/ (static files)
```

### Build Validation

The build must fail if:
- Any broken internal links (Docusaurus built-in check)
- Any missing images
- TypeScript compilation errors in custom components

## CI/CD -- Documentation Workflow

File: `.github/workflows/docs.yml`

```yaml
name: Deploy Documentation

on:
  push:
    branches: [main]
    paths:
      - 'website/**'

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
          cache-dependency-path: website/package-lock.json
      - run: cd website && npm ci
      - run: cd website && npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: website/build

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

The site deploys to `https://moukrea.github.io/cairn/`.

Deployment trigger: push to `main` that modifies files under `website/`.

GitHub repository setting required: Pages source set to "GitHub Actions".
